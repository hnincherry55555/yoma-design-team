# Journey Map Format

One map per primary segment. Columns are stages; rows are:

| Row | Content |
|-----|---------|
| Stage | Named phase (e.g. Discover → Sign up → KYC → First booking → Trip → Payment) |
| Actions | What the user does, from evidence |
| Touchpoints | App screens, calls, physical locations |
| Thoughts | Verbatim or closely paraphrased quotes |
| Emotion | Curve value -2..+2 with the driving reason |
| Pain points | Traced to evidence (participant IDs) |
| Opportunities | Phrased neutrally, not as solutions |

Rules: current state only (no aspirational future flows here); every pain point carries evidence references; peak and end moments flagged (peak-end rule).
