# WCAG 2.1 AA Checklist (design-relevant criteria)

## Perceivable
- [ ] 1.1.1 Non-text content has text alternatives (icons, images have labels)
- [ ] 1.3.1 Info and relationships not conveyed by layout alone (headings, labels defined)
- [ ] 1.3.4 Orientation not locked without reason
- [ ] 1.4.1 Color never the only indicator (errors also have icon/text)
- [ ] 1.4.3 Contrast: 4.5:1 text, 3:1 large text
- [ ] 1.4.4 Text resizable to 200% without loss
- [ ] 1.4.10 Reflow at 320px width without horizontal scroll
- [ ] 1.4.11 Non-text contrast 3:1 (component boundaries, focus indicators)
- [ ] 1.4.12 Text spacing adjustable without breakage
- [ ] 1.4.13 Content on hover/focus dismissable and persistent

## Operable
- [ ] 2.1.1 All functionality keyboard-operable
- [ ] 2.1.2 No keyboard traps
- [ ] 2.4.3 Focus order preserves meaning
- [ ] 2.4.4 Link purpose clear from text
- [ ] 2.4.6 Headings and labels descriptive
- [ ] 2.4.7 Focus visible on every interactive element
- [ ] 2.5.3 Visible label part of accessible name
- [ ] 2.5.5 (best practice) Touch targets 44x44px

## Understandable
- [ ] 3.1.1 Page language set (mind EN/MM mixed screens)
- [ ] 3.2.1 Focus does not trigger unexpected context change
- [ ] 3.3.1 Errors identified in text
- [ ] 3.3.2 Labels or instructions for all inputs
- [ ] 3.3.3 Error suggestion provided
- [ ] 3.3.4 Legal/financial submissions reversible, checked, or confirmed (Easy Lease, payments)
